## 🏗️ Architecture Highlight — Smart Bus Backend

```
Driver App
   → WebSocket (driver:location)
   → Socket.IO Server
   → Redis Stream (trip-locations)
   → Worker (Consumer Group)
       → Batch insert into PostgreSQL
       → Throttled broadcast every 2s
   → Passenger Clients (Live Map Updates)
```

> **Why it matters:** This architecture handles high-frequency GPS pings without hammering the DB — using Redis as a buffer and consumer groups for fault-tolerant processing. A real production-grade system design from day one.
